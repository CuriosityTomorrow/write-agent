# 人机协同写作最佳实践

> 基于 Write Agent 架构的网文写作工作流指南

## 核心理念

**人负责创意决策，AI 负责细节填充。**

人脑擅长：灵感、判断力、审美、大方向把控
AI 擅长：结构化、细节展开、一致性维护、大量文本生成

目标：人只输入 10% 的核心创意信息，AI 补全剩余 90% 的结构和细节，实现 5-10 倍写作效率。

---

## 阶段一：从碎片到骨架（30 分钟）

### 你手上有什么

通常你只有零散的灵感碎片：

```
末日求生、异能、校园
主角是个废物但有隐藏血脉
有个冷艳学姐后来变成队友
想写那种一步步建立据点的爽感
```

### 怎么做

**Step 1: 创建小说 → CreateWizard**

进入创建向导，在第一步输入你的关键词和碎片想法。不需要完整，能想到什么写什么。

**Step 2: AI 生成完整方案 → `generateIdea`**

系统会基于你的碎片生成完整的小说设定：标题、世界观、力量体系、主角设定、核心冲突、金手指等。

**Step 3: 逐字段微调 → `regenerateField`**

这是最关键的一步。对每个字段：
- 满意 → 跳过
- 不满意 → 点 regen 按钮，可以附带修改建议
- 例如：世界观生成了"修仙世界"，你输入"我想要末日废土，不是修仙" → AI 重新生成

**人的工作量：** 审阅 + 几句修改建议（~10 分钟）
**AI 的工作量：** 生成完整设定的所有字段（等价人工 1-2 小时）

### 技巧

- 不要试图在脑子里想清楚所有设定再开始，先让 AI 生成一版，在它的基础上改比从零开始快得多
- 关键词越具体越好："末日求生 + 异能觉醒 + 校园起点" 比 "末日" 好
- 重点关注：核心冲突、金手指、主角身份——这三项决定了整本书的走向

## 阶段二：大纲生成（20 分钟）

### 怎么做

**Step 1: 生成大纲 → `generateOutline`**

在小说详情页点击生成大纲。AI 会基于你的设定生成：
- 故事背景
- 角色列表（含性格标签、行为准则、说话风格）
- 主线剧情弧线
- 分阶段 plot_points（含章节范围、核心冲突、伏笔计划）

**Step 2: 审阅和调整**

- 检查 plot_points 的节奏是否合理
- 需要长期铺垫的大事件标记为 `event_scale: "major"`
- 调整角色的 `behavior_rules`（绝对会做/绝对不做）
- 这些规则会在后续每章生成时自动注入 prompt

**人的工作量：** 审阅大纲 + 微调角色规则（~20 分钟）
**AI 的工作量：** 从设定推导完整大纲和角色体系（等价人工 3-5 小时）

### 技巧

- `personality_tags` 最多 2 个，强迫 AI 抓住角色核心（"隐忍" + "重情义" 比 5 个标签更有效）
- `absolute_dont` 是防止 AI 写崩角色的关键，比如韩立的 "绝不为了义气冲动行事"
- 大纲不需要一次到位，后续可以随时通过 `regenerateField` 调整单个 plot_point

## 阶段三：逐章写作（每章 10-15 分钟）

### 工作流

```
对每一章：
1. [可选] 设置章节类型（setup/transition/climax），不设则系统自动分配
2. [可选] 填写本章要点/方向
3. 点击生成 → AI 流式输出章节内容
4. 审阅 → 满意则提取 intel → 进入下一章
          → 不满意则附带修改建议重新生成
```

### 系统在背后做了什么

每次生成章节时，系统自动组装上下文（你不需要操心）：

| 层级 | 内容 | 作用 |
|------|------|------|
| P0 | 小说骨架（设定、大纲） | 确保方向一致 |
| P1 | 必要角色（本章涉及的） | 确保角色不崩 |
| P2 | 前 2 章全文 | 确保文风和剧情衔接 |
| P3 | 活跃伏笔 | 确保伏笔被推进 |
| P4 | 前 3-5 章 intel | 近期剧情细节 |
| P5 | 前 6-15 章摘要 | 中期剧情脉络 |
| P5.5 | 前 16-30 章关键事件 | 关键事件不遗忘 |
| P6 | 可选角色 | 配角信息备查 |
| P7 | 卷/弧/全局摘要 | 千章不忘前文 |
| 节奏指令 | 章节类型约束 | 控制事件密度 |

### 技巧

- **每章只做一件大事**。系统的节奏控制会确保 setup 章只有 1 个主事件 + 充分展开，不要手动塞太多内容
- **善用 intel 提取**。每章写完后提取 intel，这是记忆系统的燃料。不提取 = 系统会逐渐"遗忘"
- **不满意就带建议 regen**，比如"对话太生硬，让角色更含蓄" → AI 会保留剧情但调整表达
- **climax 章可以手动指定**，不用等 6 章周期。遇到大事件爆发，直接设 chapter_type = "climax"

## 阶段四：长篇维护（每 30 章 ~15 分钟）

### 自动触发

- **每 30 章**：系统自动生成卷摘要，压缩近 30 章的情节、角色状态、关系变化
- **每 150 章**：系统自动生成弧摘要，进一步压缩

你不需要手动触发，但建议每 30 章检查一次生成的卷摘要是否准确。

### 大事件规划

当你感觉"差不多该来一个大事件了"（通常 50-200 章积累后）：

1. **选定章节范围** → 调用范围摘要，快速回顾这段剧情
2. **AI 建议大事件** → 基于摘要生成 2-3 个大事件创意
3. **你选择或修改** → 确定大事件方案
4. **AI 生成铺垫计划** → 自动规划未来 N 章的伏笔埋设
5. **写入大纲** → 扩展 plot_point，后续章节自动感知

### 技巧

- 大事件的铺垫期建议 20-50 章，太短读者没感觉，太长会忘记
- 不要同时运行超过 2 个大事件的铺垫线，否则单章信息量过大
- 利用范围摘要功能回顾之前的内容，比重读原文快 10 倍

## 常见陷阱

### 1. 过度干预
不要试图控制每一个细节。你写的"建议"越具体，AI 的发挥空间越小，效率越低。

**反例：** "让主角先往左走，遇到一个老头，老头说了一句话，然后主角往右走"
**正例：** "主角在这里获得一个关键线索，推动他前往下一个目的地"

### 2. 不提取 intel
写完章节不点 intel 提取 = 后续章节不知道前面发生了什么。这是最常见的导致剧情不连贯的原因。

### 3. 忽略角色规则
大纲生成的角色 `behavior_rules` 如果不检查和修正，角色容易写崩。花 5 分钟确认每个主要角色的"绝对不做"清单，后续省下大量返工。

### 4. 大事件没有铺垫
突然插入大事件 = 读者觉得突兀。使用大事件规划功能，让系统提前 20+ 章开始埋伏笔。

## 推荐节奏

- **日产量目标：** 6-10 章/天（每章 3000 字 = 18000-30000 字/天）
- **每章耗时：** 生成 3-5 分钟 + 审阅/微调 5-10 分钟
- **每 30 章暂停：** 检查卷摘要、回顾角色状态、规划下一阶段（~15 分钟）
- **每 100 章暂停：** 考虑是否需要规划大事件（~30 分钟）
